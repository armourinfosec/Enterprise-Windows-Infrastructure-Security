# 8.3 Filename (Short File Name)

## Overview

The **8.3 filename** (also called **Short File Name (SFN)**) is a legacy naming convention introduced in DOS and retained in Windows for backward compatibility. The format uses up to **8 characters** for the filename plus **3 characters** for the extension, and modern Windows generates these short names automatically unless the feature is disabled.

## Concepts

Older MS-DOS and 16-bit Windows applications only supported the 8.3 format. Modern Windows generates short names (unless disabled) so these legacy applications can still access files.

| Long Filename | 8.3 Filename |
| --- | --- |
| `Program Files` | `PROGRA~1` |
| `Program Files (x86)` | `PROGRA~2` |
| `Documents and Settings` | `DOCUME~1` |
| `My Document.txt` | `MYDOCU~1.TXT` |

### How Windows Creates an 8.3 Name

A long name such as:

```text
Very Long File Name.txt
```

becomes:

```text
VERYL~1.TXT
```

Generation rules:

- Spaces are removed.
- Invalid characters are omitted.
- The name is converted to uppercase.
- The first six valid characters are kept.
- `~1`, `~2`, etc., are appended if necessary.
- The extension is truncated to three characters.

## Commands

### Check Whether 8.3 Names Are Enabled

```cmd
fsutil 8dot3name query
```

Example output:

```text
The registry state is: 2 (Per volume setting)
The volume state is: 0 (8dot3 name creation is enabled)
```

### Enable or Disable Globally

Enable:

```cmd
fsutil 8dot3name set 0
```

Disable:

```cmd
fsutil 8dot3name set 1
```

Possible values:

| Value | Description |
| --- | --- |
| `0` | Enable 8.3 name creation on all volumes |
| `1` | Disable 8.3 name creation on all volumes |
| `2` | Per-volume setting (default on many systems) |
| `3` | Disable on all volumes except the system volume |

### Check a Specific Volume

```cmd
fsutil 8dot3name query C:
```

### Enable or Disable on a Specific Volume

Enable:

```cmd
fsutil 8dot3name set C: 0
```

Disable:

```cmd
fsutil 8dot3name set C: 1
```

### View the 8.3 Filename

Command Prompt:

```cmd
dir /x
```

Example:

```text
08/12/2025  10:00 AM    <DIR>          PROGRA~1     Program Files
08/12/2025  10:00 AM    <DIR>          USERS~1      Users
```

PowerShell:

```powershell
Get-ChildItem | Format-Table Name, PSChildName
```

## Best Practices

- Disable 8.3 filename creation on new servers if legacy applications do not depend on it.
- Verify compatibility before disabling on existing systems.
- Use `dir /x` only for troubleshooting or compatibility verification.
- Periodically review the setting as part of server hardening.

> [!NOTE]
> Disabling 8.3 name creation only affects newly created files and directories. Existing short names remain until they are explicitly removed or the files/directories are recreated.

## Security Considerations

Attackers sometimes enumerate 8.3 filenames to discover hidden files and directories. This can aid in:

- Information disclosure
- Directory enumeration
- Bypassing access restrictions in poorly configured applications
- Supporting Local File Inclusion (LFI) or path discovery attacks

For this reason, many organizations disable 8.3 name creation on servers where legacy application compatibility is not required.

## Troubleshooting

| Symptom | Likely cause | Resolution |
| --- | --- | --- |
| `dir /x` shows no short names | 8.3 creation disabled or names already stripped | Query with `fsutil 8dot3name query`; short names are only created for new files |
| Legacy application cannot find files after hardening | 8.3 name creation disabled on the volume | Re-enable with `fsutil 8dot3name set 0` and recreate affected files |
| Setting change appears to have no effect | Change applies only to newly created files | Recreate or rename existing files/directories to regenerate short names |
